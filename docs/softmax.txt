Softmax 详解
================================================================================

一、定义
--------------------------------------------------------------------------------

  对行向量 x = [x_0, x_1, ..., x_{w-1}]，softmax 对每个元素的计算公式为：

    softmax(x_i) = exp(x_i) / Σ_j exp(x_j)

  本项目实现（数值稳定写法，见第四节）：

    softmax(x_i) = exp(x_i - max(x)) / Σ_j exp(x_j - max(x))

================================================================================
二、性质一：所有元素之和恒为 1
================================================================================

  设 S = Σ_j exp(x_j)，则：

    Σ_i softmax(x_i) = Σ_i [ exp(x_i) / S ]
                      = (1/S) · Σ_i exp(x_i)
                      = S / S
                      = 1

  无论输入 x 取何值，分子之和与分母完全相同，结果恒为 1。

  同时 exp(x_i) > 0 恒成立，因此每个输出元素都落在 (0, 1) 区间内。

================================================================================
三、性质二：指数放大差异——大的更大，小的更小
================================================================================

  exp 是单调递增的凸函数，将线性差距放大为指数比率：

    x_a - x_b = d（线性差）
    exp(x_a) / exp(x_b) = exp(d)（指数比）

  差距 d 越大，exp(d) 增长越快，强者越强，弱者越弱。

【示例一：x = [1.0, 2.0, 3.0]】

  线性归一化（直接除以和）：
    和 = 6
    结果 = [1/6, 2/6, 3/6] ≈ [0.167, 0.333, 0.500]
    最大值 3.0 占比：50%

  Softmax：
    exp(x) = [e^1, e^2, e^3] ≈ [2.718, 7.389, 20.086]
    和 ≈ 30.193
    结果 ≈ [0.090, 0.245, 0.665]
    最大值 3.0 占比：66.5%

  Softmax 使最大值的权重从 50% 提升到 66.5%，其余值被进一步压低。

  | 方法       | 结果                          | 最大值占比 |
  |------------|------------------------------|-----------|
  | 线性归一化  | [0.167, 0.333, 0.500]        | 50%       |
  | Softmax    | [0.090, 0.245, 0.665]        | 66.5%     |

【示例二：x = [1.0, 2.0, 10.0]（极端情况，赢者通吃）】

  exp(10) ≈ 22026 >> exp(1) ≈ 2.718, exp(2) ≈ 7.389
  结果 ≈ [0.0001, 0.0003, 0.9997]

  最大值几乎独占全部权重，其余被压制到接近 0。

  直觉：softmax 是在做"赢者通吃"的竞争，
        在神经网络分类中，使模型能输出"有把握"的概率分布。

================================================================================
四、性质三：输出构成概率分布
================================================================================

  概率分布的两个必要条件：

  | 条件              | Softmax 是否满足 | 原因                    |
  |-------------------|----------------|------------------------|
  | 每项 p_i 属于(0,1) | 是             | exp(x_i) > 0 恒成立     |
  | Σ_i p_i = 1      | 是             | 性质一已证               |

【应用场景：神经网络分类】

  神经网络最后一层输出 logits（任意实数，无约束）：
    logits = [2.1, 0.5, 3.7]

  经 softmax 变换为概率分布：
    softmax → [0.15, 0.03, 0.82]

  argmax → 类别 2（概率 82%）作为预测结果

  softmax 将无约束的实数输出（logits）映射为可解释的概率分布，
  是分类网络最后一层的标准操作。

【大模型（LLM）最后一层的 softmax】

  大模型的任务是：给定已有 token 序列，预测下一个 token 的概率分布。

  ── 维度 ────────────────────────────────────────────────────────────────────

  输入（logits，线性层输出）：[batch_size, seq_len, vocab_size]
    batch_size : 并行处理的样本数（推理时通常为 1）
    seq_len    : 当前输入序列的 token 数（上下文长度）
    vocab_size : 词表大小（GPT-2: 50,257 / LLaMA-3: 128,256）

  输出（概率）：[batch_size, seq_len, vocab_size]
    形状与输入完全相同，每个元素变为 (0, 1) 内的概率值

  ── 对哪个维度归一化 ────────────────────────────────────────────────────────

  沿 dim=-1（vocab_size 维度）做 softmax：
    对序列中每个位置，独立地将该位置的 vocab_size 个 logit 变换为概率分布
    每个位置的输出满足：所有 vocab_size 个概率之和 = 1

  batch_size 和 seq_len 是"批量"维度，softmax 不跨这两个维度操作：
    batch 内各样本互不影响
    序列内各位置互不影响（每个位置独立预测下一个 token）

  ── 内存布局示意（batch_size=1, seq_len=3, vocab_size=V）──────────────────

    输入 logits（形状 [1, 3, V]）：
      位置 0：[l_0,0, l_0,1, ..., l_0,V-1]  → softmax → 概率分布（和=1）
      位置 1：[l_1,0, l_1,1, ..., l_1,V-1]  → softmax → 概率分布（和=1）
      位置 2：[l_2,0, l_2,1, ..., l_2,V-1]  → softmax → 概率分布（和=1）

    每一行是一个长度为 vocab_size 的向量，softmax 在行内归一化，
    与本项目 softmax_kernel 对 [h, w] 矩阵逐行归一化的结构完全一致：
      h = batch_size × seq_len（展平后的行数）
      w = vocab_size（每行的列数，归一化维度）

  ── 推理时只取最后一个位置 ──────────────────────────────────────────────────

  自回归生成（auto-regressive）：每次只需要预测序列最后一个位置的下一个 token

    logits[:, -1, :]           → 形状 [batch_size, vocab_size]
    softmax(logits[:, -1, :])  → 形状 [batch_size, vocab_size]，每行是一个概率分布
    argmax 或 sample           → 形状 [batch_size]，每个样本采样出一个 token id

  前面 seq_len-1 个位置的 logits 在推理时通常直接丢弃（训练时用于计算 loss）。

  ── 数量级 ──────────────────────────────────────────────────────────────────

  以 LLaMA-3（vocab_size = 128,256）为例，单次前向推理：
    softmax 输入：128,256 个 float32 = 512 KB
    需对这 128,256 个数求 max、求 exp 之和、逐元素除法
    这正是 softmax kernel 性能优化的动机：vocab_size 大，逐行操作频繁

================================================================================
五、数值稳定写法：减去行最大值
================================================================================

【问题：直接计算存在上溢】

  当 x_i 较大时，exp(x_i) → inf，导致 inf / inf = NaN。

【等价变换：对分子分母同乘 exp(-max)】

  softmax(x_i)
    = exp(x_i) / Σ_j exp(x_j)
    = [exp(x_i) · exp(-max)] / [Σ_j exp(x_j) · exp(-max)]
    = exp(x_i - max) / Σ_j exp(x_j - max)

  数学值与原公式完全相同（分子分母同比缩放，比值不变）。

【效果】

  x_i - max ≤ 0 恒成立（max 是行最大值）
    → exp(x_i - max) 属于 (0, 1]，不会上溢
    → 最大项变为 exp(0) = 1，其余项压缩到 (0, 1)

【数值示例：x = [1000, 1001, 1002]】

  | 写法           | exp(x_i) 的值         | 结果              |
  |----------------|----------------------|-------------------|
  | 直接计算        | exp(1000) = inf      | NaN               |
  | 减 max=1002    | exp(-2), exp(-1), 1  | [0.090, 0.245, 0.665] |
