PYBIND11_MODULE 与 m.def 详解
================================================================================

零、pybind11 是什么 / 如何引入
================================================================================

  pybind11 是纯头文件（header-only）的 C++ 库，无需单独编译 .so。
  功能：在 C++ 和 Python 之间建立双向绑定（函数、类、枚举、异常等）。
  底层依赖 CPython C API（Python.h），将原始 C API 封装为 C++11 风格。

  【头文件】

    直接使用 pybind11：
      #include <pybind11/pybind11.h>   // 核心：PYBIND11_MODULE、module_、cpp_function
      #include <pybind11/stl.h>        // std::vector / std::map 等 STL 类型自动转换
      #include <pybind11/numpy.h>      // numpy array 支持

    本项目只需一行，pybind11 已由 PyTorch 头文件间接引入：
      #include <torch/extension.h>     // 一站式头文件，内部已包含 pybind11

  【torch/extension.h 的展开链】

    torch/extension.h
      └─ torch/python.h
           └─ pybind11/pybind11.h    ← PYBIND11_MODULE、module_、cpp_function
                └─ Python.h          ← CPython C API（PyObject*、dlopen/dlsym 等）

  因此 interface.cpp 只写 #include <torch/extension.h>，
  PYBIND11_MODULE、m.def、pybind11::module_ 等均已可用。


一、PYBIND11_MODULE 宏
================================================================================

  PYBIND11_MODULE(TORCH_EXTENSION_NAME, m)
  {
      m.def("softmax_cuda", &softmax_cuda, "Softmax (CUDA)");
  }

  PYBIND11_MODULE 是 pybind11 提供的宏，展开后生成两样东西：
    1. CPython ABI 要求的模块入口函数 PyInit_<模块名>
    2. 用户初始化函数，花括号内的代码成为其函数体


二、参数一：TORCH_EXTENSION_NAME
================================================================================

  TORCH_EXTENSION_NAME 是 cmake target_compile_definitions 注入的宏：
    target_compile_definitions(softmax_cuda PRIVATE
        TORCH_EXTENSION_NAME=softmax_cuda   ← cmake 配置阶段写入 build.ninja
    )
    编译时展开为字面量 softmax_cuda。

  【为何必须与 .so 文件名一致】

    Python import 机制：
      import softmax_cuda
        → Python 解释器在 sys.path 中搜索 softmax_cuda.cpython-313-x86_64-linux-gnu.so
        → dlopen 加载 .so
        → 在 .so 中查找符号 PyInit_softmax_cuda（固定规则：PyInit_ + 模块名）
        → 调用该符号完成模块初始化

    PYBIND11_MODULE(softmax_cuda, m) 展开后生成：
      extern "C" PyObject* PyInit_softmax_cuda() { ... }

    若 TORCH_EXTENSION_NAME 与 .so 文件名不同：
      .so 文件名决定 Python 搜索的入口符号名
      PYBIND11_MODULE 决定实际生成的符号名
      两者不匹配 → dlopen 找不到符号 → ImportError


三、参数二：m（pybind11::module_）
================================================================================

  【m 是什么】

  m 只是一个普通的标识符（参数名），不是关键字，不是预定义符号。
  宏展开后，m 成为用户初始化函数 pybind11_init_softmax_cuda 的形参名：

    void pybind11_init_softmax_cuda(::pybind11::module_& m)
    //                                                   ↑
    //  PYBIND11_MODULE 的第二个参数决定这里的形参名
    {
        m.def("softmax_cuda", &softmax_cuda, "Softmax (CUDA)");
    }

  因此 m 可以改成任意合法 C++ 名字，语义完全不变：

    PYBIND11_MODULE(TORCH_EXTENSION_NAME, mod)
    {
        mod.def("softmax_cuda", &softmax_cuda, "Softmax (CUDA)");
    }

  【m 的类型与本质】

  m 是 pybind11::module_ 类型的引用，代表当前正在构建的 Python 模块。
  本质是对 CPython PyObject*（模块对象）的 C++ 封装，内部持有 PyObject* 指针。

  m 由 PyInit_softmax_cuda() 中的 create_extension_module 创建并以引用传入：
    auto m_obj = ::pybind11::module_::create_extension_module(...);
    pybind11_init_softmax_cuda(m_obj);   // 形参 m 绑定到 m_obj

  【m 的作用】

  向模块注册函数、子模块、类、常量等 Python 可见对象：
    通过 m.def / m.attr / m.def_submodule 等方法操作
    花括号内的所有注册操作都通过 m 完成
    m.def 每次调用都往模块内部的 __dict__ 写入一条记录


四、宏展开后的等价代码
================================================================================

  展开前（实际写法）：

    PYBIND11_MODULE(softmax_cuda, m)
    {
        m.def("softmax_cuda", &softmax_cuda, "Softmax (CUDA)");
    }

  展开后（简化版）：

    // 1. 模块元信息结构体（pybind11 内部用，存模块名、slot 等）
    static ::pybind11::module_::module_def pybind11_module_def_softmax_cuda;

    // 2. CPython ABI 入口函数，Python import 时 dlopen 后调用此符号
    //    函数名规则：PyInit_ + 模块名（固定，不可更改）
    //
    //    extern "C" 的作用：
    //      C++ 编译器默认对函数名做 name mangling（名字修饰），
    //      将参数类型编码进符号名以支持函数重载，例如：
    //        void foo(int)  → _Z3fooi
    //        void foo(double) → _Z3food
    //      修饰后的符号名是编译器私有格式，不同编译器结果不同。
    //
    //      extern "C" 告诉编译器：此函数用 C 链接约定，禁止 name mangling，
    //      符号名严格等于函数名本身：
    //        extern "C" PyObject* PyInit_softmax_cuda()
    //          → 符号名保持 PyInit_softmax_cuda（不加任何修饰）
    //
    //      Python 解释器在 dlopen 加载 .so 后，用 dlsym 按固定字符串查找入口：
    //        dlsym(handle, "PyInit_softmax_cuda")
    //      若符号被 name mangling 修饰，dlsym 找不到该字符串 → ImportError。
    //      extern "C" 保证符号名与 dlsym 查找的字符串完全一致。
    // 调用方：CPython importlib（解释器内部 C 代码），用户代码不直接调用此函数
    //   import softmax_cuda
    //     → dlopen("softmax_cuda...so")           // 操作系统加载 .so
    //     → init_fn = dlsym(handle, "PyInit_softmax_cuda")  // 按字符串查找函数地址
    //     → PyObject* mod = init_fn()             // 调用函数指针，进入此函数体
    //     → sys.modules["softmax_cuda"] = mod     // 解释器保存返回的模块对象
    extern "C" PyObject* PyInit_softmax_cuda()
    {
        // 分配模块 PyObject，__name__ = "softmax_cuda"，__dict__ 初始为空
        auto m = ::pybind11::module_::create_extension_module(
            "softmax_cuda",                      // 模块名字符串
            nullptr,                             // docstring（此处为空）
            &pybind11_module_def_softmax_cuda    // 元信息结构体
        );
        try {
            pybind11_init_softmax_cuda(m);   // 调用用户初始化函数，填充 __dict__
            return m.ptr();                  // 取出裸 PyObject*，交还给解释器
        } catch (...) {
            return nullptr;
        }
    }

    // 3. 用户花括号内的代码成为此函数的函数体
    //    由 PyInit_softmax_cuda 调用，m 是对上面 create_extension_module 结果的引用

    void pybind11_init_softmax_cuda(::pybind11::module_& m)
    {
        // m.def 在模块 __dict__ 中写入一条记录：
        //   __dict__["softmax_cuda"] = <包装了 &softmax_cuda 的 pybind11 可调用对象>
        // m.def 返回 module_& 自身，支持链式调用，此处只注册一个函数故不链式
        m.def("softmax_cuda", &softmax_cuda, "Softmax (CUDA)");
    }

完整调用过程（从 import 到函数注册完毕）：

[Python 层]
  import softmax_cuda as cuda
      │
[CPython importlib，C 层]
      ├─ dlopen("softmax_cuda...so")
      │     操作系统把 .so 映射到进程地址空间
      │
      ├─ init_fn = dlsym(handle, "PyInit_softmax_cuda")
      │     在 .so 符号表中按字符串查找函数地址
      │     extern "C" 保证符号名未被 name mangling 修饰，查找成功
      │
      └─ PyObject* mod = init_fn()
            │
[PyInit_softmax_cuda，C++ 层]
            ├─ create_extension_module("softmax_cuda", ...)
            │     分配模块 PyObject，__dict__ 初始为空
            │
            ├─ pybind11_init_softmax_cuda(m)
            │     │
[pybind11_init_softmax_cuda，C++ 层]  ← 用户花括号内的代码在这里执行
            │     └─ m.def("softmax_cuda", &softmax_cuda, "Softmax (CUDA)")
            │           生成 pybind11 函数包装对象
            │           写入：__dict__["softmax_cuda"] = <包装对象>
            │           返回 module_& 自身（副作用已完成）
            │
            └─ return m.ptr()
                  取出裸 PyObject*，返回给 CPython

[CPython importlib，C 层]
      sys.modules["softmax_cuda"] = mod   // 保存模块对象
      cuda = mod                           // import as cuda 赋值

至此 import 完成，cuda.__dict__ == {"softmax_cuda": <包装对象>, ...}


五、m.def 三个参数
================================================================================

  m.def("softmax_cuda",   &softmax_cuda,   "Softmax (CUDA)");
  //     ↑ Python 侧名字   ↑ C++ 函数指针   ↑ docstring

  【参数一："softmax_cuda"】

    注册后 Python 侧通过此名字调用：
      import softmax_cuda as cuda
      cuda.softmax_cuda(x)    ← 用的就是这个字符串

    C++ 函数名与 Python 函数名不必相同，完全由此字符串决定。
    m.def 将其写入模块的 __dict__["softmax_cuda"]。

  【参数二：&softmax_cuda】

    取 C++ 函数的地址：
      torch::Tensor softmax_cuda(torch::Tensor x) { ... }

    pybind11 拿到函数指针后，自动推断参数类型（torch::Tensor）和返回类型
    （torch::Tensor），生成一个包装对象负责类型转换。

  【m.def 的返回值】

    m.def 返回 module_&（模块自身的引用），支持链式注册：
      m.def("foo", &foo, "").def("bar", &bar, "");

    真正的作用是副作用：在模块的 __dict__ 里写入一条记录：
      module.__dict__["softmax_cuda"] = <pybind11 包装的 C++ 函数对象>

  【参数三："Softmax (CUDA)"】

    docstring，Python 查询时可见：
      help(cuda.softmax_cuda)
      # softmax_cuda(x: torch.Tensor) -> torch.Tensor
      # Softmax (CUDA)    ← 这里显示

五、m.ptr
================================================================================
  【m.ptr()】

    m.ptr() 返回 pybind11::module_ 内部持有的裸 PyObject* 指针：

      pybind11::module_ m
          └─ PyObject* ptr_      ← m.ptr() 返回这个
                 ├─ ob_type = PyModule_Type
                 └─ md_dict = {"softmax_cuda": <函数包装对象>}

    PyInit_softmax_cuda() 的返回类型是 PyObject*（CPython ABI 要求），
    m.ptr() 从 C++ 封装中取出裸指针，交还给 Python 解释器：
      return m.ptr();   // 解释器收到后把它作为 import 语句的模块对象


六、调用时序（逐步说明）
================================================================================

  ── 阶段一：import softmax_cuda ──────────────────────────────────────────────

  # Python 代码
  import softmax_cuda as cuda

  步骤 1：Python 解释器在 sys.path 中搜索 .so 文件
    搜索目标：softmax_cuda.cpython-313-x86_64-linux-gnu.so
    找到后调用：handle = dlopen("softmax_cuda...so", RTLD_NOW | RTLD_GLOBAL)

  步骤 2：dlsym 查找入口符号
    init_fn = dlsym(handle, "PyInit_softmax_cuda")
    // 固定规则：PyInit_ + 模块名
    // extern "C" 保证符号名未被 C++ name mangling 修饰

  步骤 3：调用入口函数 PyInit_softmax_cuda()

    extern "C" PyObject* PyInit_softmax_cuda()
    {
        // 3a. 分配模块 PyObject，__dict__ 为空
        auto m = ::pybind11::module_::create_extension_module(
            "softmax_cuda", nullptr, &pybind11_module_def_softmax_cuda
        );

        // 3b. 调用用户初始化函数，填充 __dict__
        pybind11_init_softmax_cuda(m);

        // 3c. 取出裸 PyObject* 返回给解释器
        return m.ptr();
    }

  步骤 4：pybind11_init_softmax_cuda(m) 执行

    void pybind11_init_softmax_cuda(::pybind11::module_& m)
    {
        m.def("softmax_cuda", &softmax_cuda, "Softmax (CUDA)");
        // 生成 pybind11 函数包装对象，写入：
        // m.__dict__["softmax_cuda"] = <包装了 &softmax_cuda 的可调用对象>
    }

  步骤 5：解释器收到 PyObject*，作为模块对象赋给 Python 变量 cuda
    cuda = <PyObject* 指向的模块>
    cuda.__dict__ == {"softmax_cuda": <函数包装对象>, ...}

  ── 阶段二：cuda.softmax_cuda(x) ────────────────────────────────────────────

  # Python 代码
  x   = torch.rand(1024, 4096, device='cuda')
  out = cuda.softmax_cuda(x)

  步骤 6：属性查找
    cuda.softmax_cuda
    → 查 cuda.__dict__["softmax_cuda"]
    → 取出 pybind11 函数包装对象（cpp_function）

  步骤 7：pybind11 包装对象被调用，执行类型转换（输入）
    Python tensor x（PyObject*）
    → pybind11 提取其 TensorImpl 指针
    → 包装为 torch::Tensor 句柄（8 bytes，ref_count++）
    GPU 显存不动

  步骤 8：pybind11 cpp_function 包装对象调用 C++ 函数
    调用方：步骤 7 中的 pybind11 cpp_function 包装对象
      类型转换完成后，包装对象持有 &softmax_cuda 函数指针，直接调用：
        cpp_function::call() → (*func_ptr)(torch::Tensor x)
      用户代码（Python 层）不直接触及 C++ 函数指针，由包装对象代劳

    torch::Tensor softmax_cuda(torch::Tensor x)
    {
        return softmax_cu(x);   // 透传到 run_kernels.cu
    }

  步骤 9：softmax_cu(x) 执行
    torch::Tensor softmax_cu(torch::Tensor x)
    {
        auto out = torch::empty_like(x);    // 分配输出 GPU 显存
        // AT_DISPATCH_FLOATING_TYPES 展开 switch，选择 float/double 分支
        AT_DISPATCH_FLOATING_TYPES(x.scalar_type(), "softmax_cuda", ([&] {
            softmax_kernel8<scalar_t><<<grid_size, block_size>>>(
                x.data_ptr<scalar_t>(),     // 裸 GPU 指针（输入）
                out.data_ptr<scalar_t>(),   // 裸 GPU 指针（输出）
                totalRow, totalCol
            );
        }));
        return out;   // 返回输出 tensor 句柄
    }

  步骤 10：GPU kernel 异步执行
    softmax_kernel8<float> 在 GPU 上并行计算：
      每个 block 处理一行，warp shuffle 规约 max 和 sum，写出归一化结果

  步骤 11：pybind11 包装对象执行类型转换（输出）
    torch::Tensor out（C++ 句柄）
    → pybind11 提取 TensorImpl 指针，包装为 Python tensor（PyObject*）
    → ref_count++，GPU 显存不动

  步骤 12：Python 收到返回值
    out = cuda.softmax_cuda(x)
    // out 是 Python tensor，与 C++ 侧 out 共享同一块 GPU 显存
    // C++ 侧局部变量 out 析构，ref_count--，但 Python 侧 out 仍持有引用
    // GPU 显存继续存活，直到 Python 侧 out 也被回收


七、torch::Tensor 的零拷贝传递
================================================================================

  pybind11 处理 torch::Tensor 时不复制 GPU 数据：

    torch::Tensor 是轻量句柄（8 bytes），内部只有一个成员：
      intrusive_ptr<TensorImpl> impl_;

    Python 侧 tensor 与 C++ 侧 torch::Tensor 共享同一个 TensorImpl，
    即共享同一块 GPU 显存（Storage::data_ptr_）。

    类型转换只涉及：
      Python 对象头 → 提取 TensorImpl 指针 → 包装为 torch::Tensor 句柄
      ref_count++ / ref_count--（原子操作，O(1)）

    GPU 显存全程不动，无任何数据拷贝。


八、TORCH_EXTENSION_NAME vs 模块名字符串一致性
================================================================================

  三处名字必须完全一致，否则 import 失败：

    ① cmake add_library 目标名 / set_target_properties 输出文件名
        → 决定 .so 文件名：softmax_cuda.cpython-313-...so

    ② target_compile_definitions TORCH_EXTENSION_NAME=softmax_cuda
        → 注入宏，PYBIND11_MODULE 展开时用于生成 PyInit_softmax_cuda 符号

    ③ Python import softmax_cuda
        → 解释器按此名查找 .so 并查找 PyInit_softmax_cuda 符号

    ① 决定 .so 文件名和 Python 搜索的入口符号名
    ② 决定 .so 内实际生成的入口符号名
    ① = ② 才能让 Python import 成功找到符号并调用
